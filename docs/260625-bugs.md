# Current Bugs

Testing the application surfaced the following issues: 

1. Nachdem ein Ring vernichtet wurde, soll beim nächsten Neufang der selben Ringgröße die Nummer inkrementiert werden. Wenn V1008 vernichtet wird, wird aktuell beim nächsten Neufang erneut V1008 vorgeschlagen.
2. Die Capslock warnung funktioniert nicht richtig. 1) Sie erkennt eine aktive feststelltaste nicht beim neuen seitenladen. 2) Unabhängig ob Capslock ein oder ausgeschalten wird, wird die warnung aktiv. 3) Die warnung geht nicht mehr weg, wenn sie einmal aktiviert wurde, auch wenn capslock gedrückt wird. 
3. Wenn ich im /data-entries view das projekt switche, wird die Liste der letzten fänge nicht aktualisiert, sondern bleibt bei der Liste des letzten Projekts. 
4. Die Breite des Ringgröße-Input-Felds kann noch deutlich verringert werden. Es soll nur so groß sein, wie notwendig um die max zwei Buchstaben anzeigen zu können. 
5. Die Autocomplete-funktion soll auch nach substrings innerhalb des string suchen: "amsel" soll auch "Wasseramsel" und "vernichtet" soll auch "Ring Vernichtet" als teil der Ergebnismenge haben.
6. Der "Ring Vernichtet" Button soll nach unten links neben die "Zurücksetzen" - "Erstellen" Buttons wandern. 
7. Der "+ Neuer Fang" Button in der nav-bar soll nicht sichtbar sein während der Nutzer auf der /data-entry page befindet.
