## `sample_iwm_illmitz.xlsx` — synthetic stand-in for the real IWM export

Generated by `generate_sample_iwm.py`. **Not real data, no personal data** — 343
invented captures from a plausible reed-bed operation at Illmitz / Neusiedlersee
(Burgenland), caught with Japannetz (`Fangmethode M`) across two summer+autumn
seasons (2023–2024). It stands in for the *pre-anonymisation* real export until
that arrives, so we can build and test the importer + anonymiser now.

It intentionally carries the shape the anonymiser must collapse: **5 Beringer**
(→ curated 2), **3 sub-sites** (→ curated 1), sequential ring runs, **40
Wiederfänge that reuse an Erstfang's ring identity** (dated later), and **3
Sonderarten** (2× Aves ignota with a mandatory Bemerkung, 1× Ring Vernichtet with
no bird data).

Regenerate (deterministic — fixed seed ⇒ byte-stable):

```bash
backend/.venv/bin/python backend/birds/demo/generate_sample_iwm.py
```

### ⚠️ Importer note — authentic IWM ≠ our export

This file mirrors the **authentic** IWM `Datenmeldung` conventions from the export
template's own example rows, which differ from what our `iwm_export.py` currently
writes. The Phase-1 importer must map the authentic codes, not just invert the
export:

| Column | Authentic IWM (this file) | Our `iwm_export.py` writes |
|--------|---------------------------|----------------------------|
| Geschlecht | letters `U` / `M` / `W` | integer `0` / `1` / `2` |
| Alter | integer `1`–`6` | integer `1`–`6` (same) |
| Fett, Muskel, Intensität, Handschwingen, Netz | text codes | numbers |
| Zusatzmarkierung | `ZZ` | blank (deferred) |
| Ringstatus | `E` / `W` (blank for Ring Vernichtet) | `E` / `W` |

Only the `Fangdaten` sheet is populated (the sheet the importer/exporter use).
